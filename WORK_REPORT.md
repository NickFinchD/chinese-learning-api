Именно это вызвало «слова не найдены» при переключении вкладок и «кнопка логин не работает». Заменил на `pgxpool.Pool` (`internal/database/postgres.go` + все 8 `repository.go`), добавил `pool.Ping` при старте. Проверено под нагрузкой: 160 параллельных запросов — все `200`, без единой ошибки (раньше падало уже на этом уровне конкурентности).

11. **Фикс критического бага: пустые списки уходили как `null`, а не `[]`.** В Go `var x []T` — это nil-слайс; `encoding/json` сериализует его как `null`. Как только `/words/saved` (нет сохранённых слов) или `/words/?search=...` (нет совпадений) возвращали 0 строк, фронтенд получал `data: null`, `store.items = null`, и любое `.length`/`.some()` на нём кидало необработанное исключение — это и ломало вкладки «намертво» до перезагрузки страницы (Vue не мог восстановиться после краша рендера). Нашёл через прокинутые в лог Vite-дев-сервера консольные ошибки браузера. Исправлено на бэкенде — `var x []T` → `x := make([]T, 0)` в `words`, `savedwords`, `courses`, `lessons`, `quizzes` (8 мест, `review` уже делал правильно). Плюс на фронте добавлен defensive-фолбэк `response.data ?? []` в `stores/savedWords.ts`, `stores/vocabulary.ts`, `stores/courses.ts` — граница внешнего API. Проверено curl: раньше `data:null`, теперь `data:[]`.

14. **Вкладка «Изучено» в словаре + расписание повторений по тестам.** Новый backend-модуль `internal/learning` с таблицей `word_learning_progress` (миграция `000014`): для каждого (user, word) хранится `stage` (0..8) и `next_eligible_at`. Правильный ответ в тренировке «Слово → перевод» продвигает слово по фиксированному расписанию: 1 (сразу) → 5 мин → 2 → 1 ч → 3 → 6 ч → 4 → 24 ч → 5 → 48 ч → 6 → 96 ч → 7 → неделя → 8 (изучено). Если правильный ответ приходит раньше, чем истёк текущий интервал, прогресс не продвигается (без штрафа); неверный ответ прогресс не откатывает. Эндпоинты: `POST /learning/:id/answer`, `GET /learning/`, `GET /learning/learned` — все под JWT. Логика расписания вынесена в `Service.RecordAnswer` с инжектируемым `now func() time.Time`, покрыта тестами (`internal/learning/service_test.go`): старт с нуля, «слишком рано», продвижение после истечения кулдауна, отсутствие отката при неверном ответе, полный проход расписания до «изучено», неизменность уже изученного слова.
    Осознанно НЕ переиспользовал таблицу `user_word_progress` модуля `review` — она уже занята другим фиче (Review, расписание 1/3/7/14/30 дней), смешивать было бы неверно.
    Frontend: `types/learning.ts`, `services/learning.ts`, `stores/learning.ts`; `VocabularyPage` получил третью вкладку «Изучено»; `wordTraining`-стор при старте исключает уже изученные слова из тренировки (с отдельным сообщением, если изучены все сохранённые), после каждого ответа шлёт результат на `/learning/:id/answer` и показывает «Этап N из 8» / «🎉 Слово изучено!».

    - `user_stats` (xp, total_seconds_active, last_heartbeat_at), `achievements` (каталог), `user_achievements` (разблокировки).
    - **Время в приложении** через heartbeat (`POST /gamification/heartbeat` каждые 30 сек с фронта, только когда вкладка видима — `document.visibilityState`). Добавляемое время = реальный wall-clock зазор с прошлого heartbeat, но не больше 90 сек за раз — так спящий комп/фоновая вкладка не надувают счётчик задним числом.
    - **XP и уровни** — треугольная прогрессия (уровень 2 = 100xp, 3 = 300xp, 4 = 600xp, ...), т.е. каждый следующий уровень требует больше опыта, как в игре.
    - **Достижения** — 3 категории × 3 тира (9 штук), ровно по образцу из запроса: «За усидчивость I/II/III» (10/50/150 часов в приложении), «Эрудит I/II/III» (10/50/150 выученных слов, через уже существующий `learning`-модуль), «Отличник I/II/III» (5/20/50 завершённых уроков, через существующий `progress`). Разблокировка пересчитывается при каждом запросе прогресса/ачивок (сравнение метрик с порогами), защищена уникальным индексом от повторного начисления XP.
    - Frontend: `stores/gamification.ts`, heartbeat-таймер в `DefaultLayout.vue`, компактный бейдж «Ур. N · XP» в шапке (кликабельный, ведёт на страницу), страница `AchievementsPage.vue` (прогресс-бар до след. уровня + карточки ачивок по категориям, разблокированные подсвечены и с датой получения).
    - Тесты: XP/уровень формула, разблокировка при достижении порога, отсутствие повторного начисления XP для уже разблокированных.
    - Проверено: heartbeat, разблокировка «Эрудит I» при 10 выученных словах (+50 XP корректно отразилось в прогрессе).

26. **Фикс: автосохранение слов при наведении было слишком агрессивным.** `@mouseenter` срабатывал мгновенно при любом прохождении курсора по тексту — включая обычное чтение и движение мыши к кнопкам «Пиньинь»/«Перевод» ниже абзаца — из-за чего почти весь текст сохранялся в «Словарь» за секунды, и казалось, что «на любое слово пишет уже сохранено» (нашёл через реальные данные в БД: у аккаунта уже было 12, потом после ручной чистки снова 7-8 сохранённых слов, совпадающих с содержимым текста). Добавил задержку 500 мс (`@mouseenter` запускает таймер, `@mouseleave` его сбрасывает) — сохранение теперь срабатывает только при осознанной задержке курсора на слове, а не при любом пролёте мыши. Очистил накопившиеся тестовые сохранённые слова из реального аккаунта пользователя.

27. **Сохранение слова из текста — теперь по клику на звёздочку, не по наведению.** Убрал авто-сохранение по hover (даже с задержкой — всё равно слишком легко срабатывало на обычном чтении). В тултипе теперь кликабельная звёздочка ☆/★ (жёлтая = уже сохранено); тултип получил `group-hover:pointer-events-auto`, чтобы кнопка была кликабельной именно во время наведения (в остальное время `pointer-events-none`, чтобы невидимый тултип не перехватывал клики по соседним словам).

28. **Фикс: подсказка со звёздочкой пропадала раньше, чем курсор до неё доходил.** Между словом и всплывающей подсказкой (позиционирована абсолютно, выше строки) есть зазор — при движении курсора туда мышь на мгновение выходит из зоны `:hover`, CSS-тултип успевает скрыться. Заменил hover-модель на клик: клик по слову открывает подсказку (держится, пока не закроешь — не зависит от положения курсора), повторный клик по слову или клик мимо (в любом месте абзаца) закрывает её; клик по звезде внутри подсказки (`@click.stop`) сохраняет/убирает слово, не закрывая саму подсказку.

29. **Дизайн, этап 1: основа (glassmorphism + светлая/тёмная тема).** Посмотрел референсы в папке `дизайн/` (глассморфизм светлый/тёмный, иллюстративный стиль, палитра `цвет.jpg`: `#E27D60` терракота, `#85DCBA` мятный, `#E8A87C` охра, `#C38D9E` мальва, `#41B3A3` тёмная бирюза).
    - **Важная находка:** установленная версия Tailwind (4.3.2, тянется из корневого `node_modules` — у `frontend/` своей копии нет, это давний пробел в `package.json`) не поддерживает `@theme`-блок с кастомными цветами так, как ожидается в доке v4 — блок протекает в вывод как обычный CSS, `bg-teal`-подобные утилити-классы не генерируются. Плюс её CSS-оптимизатор обрезает даже обычные `:root { --x: ... }` переменные как «неиспользуемые», даже если они реально указаны через `var()` внутри произвольного значения. Проверено эмпирически (тестовые классы, замер скомпилированного CSS через curl). Из-за этого все цвета палитры прописаны как литеральные hex прямо в arbitrary-value классах Tailwind (`bg-[#41b3a3]`, `bg-[#41b3a3]/40` для прозрачности) — это единственный вариант, который реально переживает сборку в этой версии.
    - **Тёмная/светлая тема** — вручную переключаемая (не только через `prefers-color-scheme`): `@custom-variant dark (&:where(.dark, .dark *))` в `style.css`, `stores/theme.ts` (сохраняет выбор в localStorage, применяет класс `.dark` на `<html>`, инициализируется в `main.ts` до первого рендера — без мигания), `components/base/ThemeToggle.vue` (🌙/☀️).
    - **Фон** — `components/layout/GlassBackground.vue`: размытые цветные пятна (`blur-3xl`) всех 5 цветов палитры поверх `bg-gray-50`/`dark:bg-slate-950`, зафиксирован на весь сайт через `App.vue`.
    - **Базовые компоненты** переведены на стекло: `BaseCard` (полупрозрачный фон + `backdrop-blur-xl` + тонкая белая обводка), `BaseButton` (пилюля, бирюзовая заливка), `BaseInput` (полупрозрачное поле, бирюзовый фокус) — все с `dark:`-вариантами.
